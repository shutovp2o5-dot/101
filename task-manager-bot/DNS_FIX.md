# Решение проблемы с DNS при работе через модем

Если бот не может подключиться к Telegram API из-за проблем с DNS, выполните следующие шаги:

## Способ 1: Настройка DNS в macOS (Рекомендуется)

1. Откройте **Системные настройки** → **Сеть**
2. Выберите ваше подключение (модем/мобильная точка доступа)
3. Нажмите кнопку **"Дополнительно"**
4. Перейдите на вкладку **"DNS"**
5. Нажмите **"+"** и добавьте следующие DNS серверы:
   - `8.8.8.8` (Google DNS)
   - `8.8.4.4` (Google DNS)
   - `1.1.1.1` (Cloudflare DNS)
   - `1.0.0.1` (Cloudflare DNS)
6. Нажмите **"OK"** и **"Применить"**
7. Перезапустите бота

## Способ 2: Использование другой сети

Если настройка DNS не помогает:
- Подключитесь к Wi-Fi сети
- Или используйте другую точку доступа
- Затем запустите бота снова

## Способ 3: Проверка подключения

Проверьте, что интернет работает:
```bash
# Проверка подключения
ping -c 3 8.8.8.8

# Проверка DNS
nslookup api.telegram.org
```

## Способ 4: Временное решение через терминал

Можно временно изменить DNS только для текущей сессии:
```bash
# Для текущей сессии (требует sudo)
sudo networksetup -setdnsservers "USB 10/100/1000 LAN" 8.8.8.8 1.1.1.1
```

Замените `"USB 10/100/1000 LAN"` на имя вашего сетевого подключения.

## Проверка после настройки

После настройки DNS запустите бота снова:
```bash
cd task-manager-bot
python3 bot.py
```

Бот должен успешно подключиться к Telegram API.
